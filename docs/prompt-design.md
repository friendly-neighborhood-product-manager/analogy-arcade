# Analogy Arcade Prompt Design

This document captures the prompt strategy for Analogy Arcade.

The product uses a multi-agent workflow:

```text
Explainer Agent
→ Analogy Critic Agent
→ Rewrite Agent
→ Final Learning Card
```

The goal is not just to generate a clever analogy. The goal is to create a useful, fun, and safer explanation by separating generation, evaluation, and improvement.

The live MVP also includes a richer frontend UX:

- Custom interest text input
- Preset interest chips
- Audience-level chips
- Multi-select output-style chips
- Progress tracker
- Input summary pills
- Interactive mini quiz
- Quota-free sample card
- Feedback capture

---

## 1. Prompt Design Principles

Analogy Arcade prompts follow these principles:

| Principle | Why it matters |
|---|---|
| Personalization | Explanations should map to the user’s selected or custom interest world |
| Consistency | Every response should follow a predictable learning-card format |
| Accuracy | Analogies should be fun without becoming misleading |
| Guardrails | The app should avoid sensitive advice, copyrighted material, and overconfident claims |
| Evaluation | A critic step should score output quality before final delivery |
| Rewrite loop | The final answer should improve on the first draft |
| Transparency | Users should see product-level workflow signals, not hidden chain-of-thought |
| Interactivity | The mini quiz should support clickable answer reveal in the frontend |
| Quota awareness | Prompts should be efficient because the MVP uses free-tier model calls |

---

## 2. Shared Input Variables

All prompts use the same core inputs.

| Variable | Description |
|---|---|
| `request_id` | Unique request identifier generated by Apps Script |
| `topic` | Topic the user wants explained |
| `interest_world` | Preset or custom interest world used for the analogy |
| `audience_level` | Intended audience level selected by the user |
| `output_style` | Plus-separated string of selected output-style chips |

Example input:

```json
{
  "request_id": "AA-20260615-123456-ABCDE",
  "topic": "OAuth",
  "interest_world": "Vacation itineraries",
  "audience_level": "Product Builder",
  "output_style": "ELI5 + analogy + mapping table + where the analogy breaks + mini quiz"
}
```

---

## 3. Current Audience Levels

The frontend sends one of these audience levels.

| Audience level | Intended behavior |
|---|---|
| `Newbie` | Use very simple language and avoid jargon |
| `Curious Learner` | Keep it beginner-friendly but include a little more context |
| `Tech Starter` | Explain technical ideas clearly without assuming deep expertise |
| `Product Builder` | Make the explanation useful for product / business decision-making |
| `Boss Mode` | Keep it concise, strategic, and executive-friendly |

In the frontend input summary, `Boss Mode` is displayed as `Boss`. The backend still receives the full value unless changed later.

---

## 4. Current Output Styles

The frontend sends selected output styles as a plus-separated string.

Default selected styles:

- `ELI5`
- `analogy`
- `mapping table`
- `where the analogy breaks`
- `mini quiz`

Optional selected styles:

- `make it funnier`
- `make it shorter`
- `include agent note`

Prompt behavior should respect these selected styles where possible, while still keeping the output useful and safe.

---

## 5. Required Output Contract

For the current frontend renderer, the generated response should use Markdown sections.

Recommended response structure:

```text
## Tiny Definition
...

## ELI5 Explanation
...

## Analogy
...

## Mapping Table
| Topic concept | Interest-world analogy |
|---|---|
| ... | ... |

## Where the Analogy Breaks
...

## Mini Quiz
Question text here?

A. Option A
B. Option B
C. Option C

**Correct answer:** A
```

The `Mini Quiz` format matters because the frontend parser turns it into an interactive clickable quiz.

Quiz formatting rules:

- Use the heading `## Mini Quiz`.
- Include one question.
- Use options labeled `A.`, `B.`, and `C.`.
- Include the answer line exactly like: `**Correct answer:** A`.
- Do not reveal the answer before the user clicks in the UI; the frontend hides/reveals it based on this format.

## 6. Explainer Agent Prompt

The Explainer Agent creates the first draft of the explanation.

```text
You are the Explainer Agent for Analogy Arcade.

Analogy Arcade explains confusing topics through worlds the user already understands.

Use the user inputs below:

Topic: {{topic}}
Interest world: {{interest_world}}
Audience level: {{audience_level}}
Output style: {{output_style}}

Create a fun, clear explanation.

Return the answer in Markdown with these sections:

## Tiny Definition
One short definition of the topic.

## ELI5 Explanation
A simple explanation for the selected audience level.

## Analogy
Explain the topic through the selected interest world.

## Mapping Table
Map 4-6 parts of the topic to parts of the interest world.

## Where the Analogy Breaks
Explain the limits of the analogy so the user does not learn the wrong thing.

## Mini Quiz
Create one multiple-choice question with 3 options.

Use this exact quiz format:

Question text?

A. First option
B. Second option
C. Third option

**Correct answer:** A

Rules:
- Keep it friendly and clear.
- Make it fun, but do not make it misleading.
- Respect the selected audience level.
- Respect the selected output style where possible.
- If the user asks for "make it shorter", keep the answer concise.
- If the user asks for "make it funnier", add light humor without reducing accuracy.
- Do not use copyrighted lyrics, movie dialogue, logos, or branded assets.
- If the topic is medical, legal, financial advice, current news, or high-stakes, include a clear caveat.
- Keep the total answer under 500 words unless the selected output style requires more detail.
```

## 7. Analogy Critic Agent Prompt

The Critic Agent evaluates whether the first draft is clear, useful, fun, and not misleading.

```text
You are the Analogy Critic Agent for Analogy Arcade.

Your job is to evaluate whether the generated explanation is clear, useful, fun, and not misleading.

User request:
Topic: {{topic}}
Interest world: {{interest_world}}
Audience level: {{audience_level}}
Output style: {{output_style}}

Generated explanation:
{{explainer_output}}

Evaluate the explanation using this rubric:

1. clarity_score: Is it easy to understand?
2. accuracy_score: Does it avoid obvious factual errors?
3. analogy_fit_score: Does the analogy map well to the topic and interest world?
4. usefulness_score: Would the user actually learn from it?
5. fun_score: Is it engaging and memorable?
6. safety_score: Does it avoid risky, misleading, or copyrighted content?

Also check:
- Does it include "Where the Analogy Breaks"?
- Does the analogy world appear meaningfully throughout the explanation?
- Does the mini quiz follow the required format?
- Does the answer match the requested audience level?
- Does the output avoid overconfident advice for high-stakes topics?

Return only valid JSON in this exact shape:

{
  "clarity_score": 0,
  "accuracy_score": 0,
  "analogy_fit_score": 0,
  "usefulness_score": 0,
  "fun_score": 0,
  "safety_score": 0,
  "overall_score": 0,
  "overall_pass": true,
  "issues": [],
  "recommended_improvements": [],
  "critic_summary": ""
}

Scoring rules:
- Scores must be integers from 1 to 5.
- overall_score should be the average of the six scores.
- overall_pass should be false if clarity_score, accuracy_score, or analogy_fit_score is below 4.
- Keep critic_summary under 40 words.
- Return JSON only. No Markdown. No explanation outside the JSON.
```

## 8. Rewrite Agent Prompt

The Rewrite Agent improves the explanation using the critic’s feedback.

```text
You are the Rewrite Agent for Analogy Arcade.

Your job is to improve the original explanation using the Analogy Critic Agent’s feedback.

User request:
Topic: {{topic}}
Interest world: {{interest_world}}
Audience level: {{audience_level}}
Output style: {{output_style}}

Original explanation:
{{explainer_output}}

Critic review:
{{critic_output}}

Rewrite the explanation so it is clearer, more accurate, more useful, and more fun.

Return the final answer in Markdown with these sections:

## Tiny Definition
One short definition of the topic.

## ELI5 Explanation
A simple explanation for the selected audience level.

## Analogy
Explain the topic through the selected interest world.

## Mapping Table
Map 4-6 parts of the topic to parts of the interest world.

## Where the Analogy Breaks
Explain the limits of the analogy so the user does not learn the wrong thing.

## Mini Quiz
Create one multiple-choice question with 3 options.

Use this exact quiz format:

Question text?

A. First option
B. Second option
C. Third option

**Correct answer:** A

## Agent Note
Briefly explain what improved from the original draft, if the user requested an agent note.

Rules:
- Keep it friendly and clear.
- Make it fun, but do not make it misleading.
- Respect the selected audience level.
- Respect the selected output style where possible.
- If "include agent note" was not requested, omit the Agent Note section unless it is useful for clarity.
- If the user selected "make it shorter", keep the answer concise.
- If the user selected "make it funnier", add light humor without reducing accuracy.
- Do not use copyrighted lyrics, movie dialogue, logos, or branded assets.
- If the topic is medical, legal, financial advice, current news, or high-stakes, include a clear caveat.
- Keep the total answer under 600 words.
```

## 9. Why the Prompts Are Split

The workflow is intentionally split into multiple prompts instead of one large prompt.

| Prompt | Product job |
|---|---|
| Explainer Agent | Generate first user value |
| Critic Agent | Evaluate the output before final delivery |
| Rewrite Agent | Improve quality using the critic’s feedback |

This separation makes the product easier to reason about, evaluate, debug, and improve.

---

## 10. Quality Rubric

The Critic Agent scores each output on six dimensions.

| Dimension | Score range | What a 5 means |
|---|---:|---|
| Clarity | 1-5 | The explanation is easy to understand |
| Accuracy | 1-5 | It avoids obvious factual errors |
| Analogy fit | 1-5 | The analogy maps well to the selected interest world |
| Usefulness | 1-5 | The user would actually learn from it |
| Fun | 1-5 | The output is engaging and memorable |
| Safety | 1-5 | It avoids misleading, risky, or copyrighted content |

A rewrite should happen when any of these are true:

```text
clarity_score < 4
accuracy_score < 4
analogy_fit_score < 4
```

The current MVP runs the Rewrite Agent every time. A future version can make this conditional.

## 11. Frontend-Dependent Prompt Requirements

Some prompt requirements exist because the frontend expects consistent output.

| Requirement | Why it matters |
|---|---|
| Markdown headings | The learning card renderer formats headings and sections |
| Markdown tables | The renderer converts mapping tables into HTML tables |
| `## Mini Quiz` heading | The renderer detects this section and converts it into an interactive quiz |
| Options labeled `A.`, `B.`, `C.` | The quiz parser uses these labels |
| `**Correct answer:** A` | The quiz parser uses this line to reveal the answer after click |
| “Where the Analogy Breaks” section | Key product guardrail |
| Concise sections | Keeps the learning card usable in the browser |

---

## 12. Guardrail Language

Every generation prompt includes rules to avoid:

- Copyrighted lyrics
- Movie dialogue
- Logos
- Branded assets
- Overconfident high-stakes advice
- Misleading analogies

The product also requires a section called:

```text
Where the Analogy Breaks
```

This is a key product guardrail because analogies are useful but incomplete.

---

## 13. Product-Level Trace vs Hidden Reasoning

Analogy Arcade should show users product-level workflow signals, such as:

```text
Explainer Agent created a first draft.
Critic Agent reviewed clarity, accuracy, analogy fit, usefulness, fun, and safety.
Rewrite Agent improved the final answer.
```

The app should not expose hidden chain-of-thought. It should expose observable workflow steps, tool calls, scores, and product decisions.

---

## 14. Current Prompt / UX Interaction

The frontend adds some value outside the model prompt:

| UX feature | Prompt dependency |
|---|---|
| Input summary pills | Uses user input directly; not generated by the model |
| Progress tracker | Uses frontend state; not generated by the model |
| Quota-free sample card | Uses a static sample output; no Gemini call |
| Interactive quiz | Requires prompt output to follow quiz format |
| Feedback form | Captures user feedback after generation |
| Back-to-top button | Navigation-only; no model dependency |
