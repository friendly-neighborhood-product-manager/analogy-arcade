# Analogy Arcade Activepieces Workflow Runbook

This document describes the Activepieces workflow powering Analogy Arcade.

The goal is to make the workflow understandable even though the actual automation lives in Activepieces.

The latest live app includes richer frontend UX in Google Apps Script:

- Custom interest text input
- Preset interest chips
- Audience-level chips
- Multi-select output-style chips
- Right-side progress tracker
- Input summary pills
- Interactive mini quiz rendering
- Quota-free sample card
- Feedback capture
- Auto-scroll and back-to-top controls

The Activepieces workflow remains focused on the AI and logging backend:

```text
Webhook request
→ Explainer Agent
→ Analogy Critic Agent
→ Rewrite Agent
→ Log request
→ Log output
→ Log agent trace
→ Return response
```

---

## 1. Workflow Name

```text
Generate Analogy Arcade Explanation
```

---

## 2. Workflow Purpose

This workflow receives a request from the Google Apps Script web app, runs the AI agent workflow, logs the request/output/trace to Google Sheets, and returns the final explanation to the browser.

Runtime pattern:

```text
Webhook request
→ Explainer Agent
→ Analogy Critic Agent
→ Rewrite Agent
→ Log request
→ Log output
→ Log agent trace
→ Return response
```

The workflow does **not** handle the quota-free sample card. The sample card is rendered locally in the Apps Script frontend and does not call Activepieces or Gemini.

---

## 3. Trigger

### Step 1: Catch Webhook

| Field | Value |
|---|---|
| Piece | Webhook |
| Trigger | Catch Webhook |
| Method | POST |
| Authentication | None |
| URL type used by Apps Script | `/sync` |

The Google Apps Script web app calls the Activepieces webhook URL stored in Apps Script Properties:

```text
ACTIVEPIECES_WEBHOOK_SYNC_URL
```

The webhook URL is intentionally not stored in GitHub.

---

## 4. Expected Request Body

Example request body from the current frontend:

```json
{
  "request_id": "AA-20260615-123456-ABCDE",
  "topic": "APIs",
  "interest_world": "Cricket",
  "audience_level": "Newbie",
  "output_style": "ELI5 + analogy + mapping table + where the analogy breaks + mini quiz"
}
```

### Input fields

| Field | Description |
|---|---|
| `request_id` | Unique ID generated by Google Apps Script |
| `topic` | Topic the user wants explained |
| `interest_world` | Preset or custom interest world |
| `audience_level` | Audience level selected through clickable chips |
| `output_style` | Plus-separated string generated from selected output-style chips |

---

## 5. Frontend Inputs That Feed the Workflow

The frontend transforms the user’s UI selections into the webhook body.

### Topic

| Frontend control | Backend field |
|---|---|
| Topic text field | `topic` |

### Interest World

| Frontend behavior | Backend value |
|---|---|
| User types custom interest | Custom text becomes `interest_world` |
| User clicks preset chip | Preset label becomes `interest_world` |
| User accepts suggestion | Suggested preset becomes `interest_world` |
| User clicks selected preset again | Interest field clears before submission |

Current preset interest worlds:

- Football tactics
- Space missions
- Bollywood drama
- Brick-by-brick builds
- Vacation itineraries
- Superhero-style teams
- Arcade games
- Cooking

### Audience Level

| Frontend control | Backend value |
|---|---|
| 🌱 Newbie | `Newbie` |
| 🔎 Curious Learner | `Curious Learner` |
| 🛠️ Tech Starter | `Tech Starter` |
| 🚀 Product Builder | `Product Builder` |
| 👑 Boss Mode | `Boss Mode` |
| No audience selected | `General audience` |

In the frontend input summary, `Boss Mode` is shortened to `Boss`. The workflow still receives the full `Boss Mode` value.

### Output Style

The frontend sends selected output-style chips as a plus-separated string.

Default selected styles:

- ELI5
- analogy
- mapping table
- where the analogy breaks
- mini quiz

Optional styles:

- make it funnier
- make it shorter
- include agent note

Example backend value:

```text
ELI5 + analogy + mapping table + where the analogy breaks + mini quiz + make it funnier
```

If no output style is selected, Apps Script defaults to:

```text
ELI5
```

## 6. Agent Steps

### Step 2: Explainer Agent

| Field | Value |
|---|---|
| Piece | Google Gemini |
| Action | Generate Content |
| Model | Gemini 2.5 Flash-Lite |
| Tool Type | Blank |
| Purpose | Generate the first explanation draft |

The Explainer Agent receives:

- Topic
- Interest world
- Audience level
- Output style

The Explainer Agent creates:

- Tiny definition
- ELI5 explanation
- Personalized analogy
- Mapping table
- Where the analogy breaks
- Mini quiz

### Recommended Explainer Output Format

The frontend Markdown renderer expects the response to include sections like:

```text
## Tiny Definition
...

## ELI5 Explanation
...

## Analogy
...

## Mapping Table
| Topic concept | Interest-world analogy |
|---|---|
| ... | ... |

## Where the Analogy Breaks
...

## Mini Quiz
Question text?

A. Option A
B. Option B
C. Option C

**Correct answer:** A
```

The `Mini Quiz` format is important because the frontend converts it into an interactive quiz.

---

### Step 3: Analogy Critic Agent

| Field | Value |
|---|---|
| Piece | Google Gemini |
| Action | Generate Content |
| Model | Gemini 2.5 Flash-Lite |
| Tool Type | Blank |
| Purpose | Score and critique the first draft |

The Critic Agent evaluates:

| Dimension | Meaning |
|---|---|
| Clarity | Is the explanation easy to understand? |
| Accuracy | Does it avoid obvious factual errors? |
| Analogy fit | Does the analogy map well to the topic and interest world? |
| Usefulness | Would the user actually learn from it? |
| Fun | Is it engaging and memorable? |
| Safety | Does it avoid risky, misleading, or copyrighted content? |

The Critic Agent should also check:

- Whether the selected interest world is used meaningfully
- Whether the answer matches the selected audience level
- Whether the mini quiz follows the expected parser-friendly format
- Whether “Where the Analogy Breaks” is included
- Whether the answer avoids overconfident advice for high-stakes topics

The Critic Agent returns a JSON-like scorecard.

---

### Step 4: Rewrite Agent

| Field | Value |
|---|---|
| Piece | Google Gemini |
| Action | Generate Content |
| Model | Gemini 2.5 Flash-Lite |
| Tool Type | Blank |
| Purpose | Improve the original explanation using critic feedback |

The Rewrite Agent returns the final output shown to the user.

Current MVP behavior:

```text
Rewrite Agent runs every time.
```

Future planned behavior:

```text
Rewrite Agent only runs when clarity, accuracy, or analogy fit score is below threshold.
```

The Rewrite Agent should preserve the expected Markdown and mini quiz format so the frontend renderer can display the response correctly.

## 8. Return Response Step

### Final Response Shape

The workflow returns JSON to the Google Apps Script web app.

```json
{
  "app": "Analogy Arcade",
  "request_id": "AA-20260615-123456-ABCDE",
  "status": "success",
  "draft_result": "First draft from Explainer Agent",
  "critic": "Critic Agent scorecard",
  "result": "Final rewritten explanation"
}
```

The Apps Script web app displays the `result` field.

The frontend also uses the user’s original inputs to display the input summary above the result card:

```text
Explain <topic> in terms of <interest world> like I'm a <audience level>
```

That input summary is created in the frontend, not by Activepieces.

## 11. Quota Notes

The workflow currently uses three Gemini calls per successful user request:

| Agent step | AI calls |
|---|---:|
| Explainer Agent | 1 |
| Analogy Critic Agent | 1 |
| Rewrite Agent | 1 |
| Total | 3 |

This means free-tier AI quotas are consumed quickly.

The Google Apps Script web app includes a daily request cap to protect free-tier usage.

Current cap:

```text
3 requests per day
```

Configured in Apps Script Properties:

```text
DAILY_REQUEST_LIMIT
```

The sample card does not use Gemini quota.

## 12. Known Issues and Fixes

### Issue: Gemini quota exceeded

Symptom:

```text
429 quota exceeded
```

Cause:

```text
Gemini free-tier request limit reached.
```

Fix:

```text
Wait for daily quota reset or reduce workflow calls.
```

Product mitigation:

```text
Daily request cap in Apps Script.
Quota-free sample card for demos.
```

---

### Issue: Gemini high demand

Symptom:

```text
503 model unavailable / high demand
```

Cause:

```text
Model temporarily overloaded.
```

Fix:

```text
Retry later or switch model if available.
```

---

### Issue: Activepieces variables do not appear

Symptom:

```text
No sample data yet
```

Cause:

```text
Webhook trigger has not received test payload in editor mode.
```

Fix:

```text
Send a POST request to the webhook `/test` URL using Hoppscotch.
```

---

### Issue: Apps Script feedback cannot write to Sheets

Symptom:

```text
You do not have permission to call SpreadsheetApp.openById.
```

Cause:

```text
Missing Google Sheets OAuth scope.
```

Fix:

```text
Add spreadsheets OAuth scope to appsscript.json and reauthorize the script.
```

---

### Issue: Mini quiz does not render interactively

Symptom:

```text
Mini quiz appears as plain text.
```

Likely cause:

```text
Model output did not follow the expected quiz format.
```

Expected quiz format:

```text
## Mini Quiz
Question?

A. Option A
B. Option B
C. Option C

**Correct answer:** A
```

Fix:

```text
Tighten Explainer and Rewrite prompts to enforce the quiz format.
Add output validation later.
```
